#document는 DOM 트리 구조의 최상위 객체
#브라우저는 HTML 문서를 로드하기 전에 document 객체를 먼저 만든다.
#document 객체는 DOM트리의 ROOT
#모든 DOM 객체 접근 경로의 시작점이다.
#document는 DOM 객체는 아니지만 document 객체이다.
#document 객체로서 프로퍼티와 속성을 지니고 있다.
#document 객체는 메서드와 컬렉션 프로퍼티가 있다.
#간단한 예로 document.write 메서드
#document 객체를 통해 현재 출력된 문서에 대한 정보를 전반적으로 알아낼 수 있다.
#document는 window의 지식이다.
#따라서 window.document와 document는 같은 표현이다.
#모든 DOM 객체는 HTML 태그로 구성되어 있고 style 객체에 접근이 가능하다.
#document객체는 style 객체가 없다.


document객체의 프로퍼티

1.location : 현재 문서의 URL 정보
2.domain : 서버의 도메인 이름
3.readyState : 문서의 로딩 상태 표시, loading -> interactive -> complete 순으로 해결
4.title 문서의 제목
5.body : 문서의 바디 객체
6.head : 문서의 헤드 객체
7.activeElement : 문서에서 현재 포커스를 받는 객체
8.URL : URL 주소


document 컬렉션

1.images : 문서 내의 모든 이미지를 배열로
2.links : 문서 내 모든 링크를 배열로
3.forms : 폼을 묶어 배열로


document 메서드

1.getElementByTagName() => 결과가 배열 =>인덱싱으로 접근
2.getElementByClassName() => 결과가 배열 =>인덱싱으로 접근
3.getElementByName() => 결과가 배열 =>인덱싱으로 접근
4.getElemnetById() => 결과가 단일 값
5.addEventListner() : document 객체에 이벤트 리스너 등록
6.close() : document 객체에 있는 HTML 컨텐츠를 브라우저에 출력하고, 더 이상 쓰기를 받지 않는다.
7.open() : document 객체에 담긴 컨텐츠를 모두 지우고, 새로운 HTML을 쓸 수 있는 상태로 열기
8.removeEventListner() : 등록되어 있는 이벤트 리스너를 삭제
9.write() : document.에 HTML 컨텐츠 삽입


HTML 문서의 동적 구성

HTML 문서가 로드 완료 후에 document에 새로운 HTML 추가 방법.
DOM객체를 로드 후에 만들어서 추가 가능
createElement로 객체를 생성한다.
생성한다고 출력되는 것은 아니다.
생성 후 DOM 트리 구조에 추가해야 출력이 된다.

#document.creatElement("div"); 함수를 호출하면 이미 로드 된 문서에 div 객체가 생성된다.

#setAttribute("속성명","속성값); 으로 동적으로 생성된 dom 객체에 대한 속성을 추가

#innerHTML로 동적으로 생성한 dom객체에 대한 컨텐츠 추가

후에 트리에 넣어줘야 함
