#DOM 객체 
document object model
HTML 태그하나하나를 전부 객체화
HTML DOM은 W3C 표준이다.
즉 모든 브라우저에서 호환성이 있다.

#BOM 객체
browser object model
브라우저 윈도우를 나타내는 객체
웹페이지의 히스토리, 스크린 객체 등

#DOM객체의 목적
HTML 태그가 출력된 모양과 컨텐츠를 제어
CSS3 스타일 프로퍼티를 변경할 수 있다.
HTML 컨텐츠, 이미지, 텍스트 등 변경할 수 있다.



#DOM트리
HTML 태그의 부모 자식 관계에 따라 DOM객체들은
트리 구조를 구성한다. 이것을 DOM 트리 라고 함
자바스크립트 언어를 통해 HTML에 접근하는 방식이
DOM트리 구조를 파악해서 접근해야 한다.


#DOM 트리의 ROOT
root : Document
Document는 DOM객체가 아니다.
Document 아래 HTML 부터 DOM객체로 인정


#DOM객체의 종류
모든 HTML태그
#HTML 태그 하나마다 하나의 DOM객체 생성

#DOM트리는 부모 자식 관계로 구성

#HTML 태그의 출력 DOM객체

1. 브라우저가 HTML 문서를 읽고
2.브라우저가 HTML로부터 DOM객체를 생성한다.
3. DOM객체의 트리구조를 생성한다.
4. HTML 태그를 화면에 출력


#DOM객체의구성
DOM객체는HTML태그의 속성이 그대로 반영

#HTML 태그의 요소 
driver.find_element_by_CSS,class

 박스모델

#DOM객체의 구성요소  코어 객체.........

1.프로퍼티
DOM객체의 멤버변수로 HTML태그의 속성을 반영한다.

2.메소드
DOM 객체의 멤버함수. HTML태그를 제어하는데 활용

3.컬렉션
정보를 집합으로 표현, 일종의 배열
children 컬렉션은 DOM객체의 모든 자식 DOM객체에 대한 주소를 가진다.

4.이벤트 리스너
 HTML태그에 작성된 이벤트 리스너를 포함한다.

5.CSS스타일
Style프로퍼티를 통해 HTML태그에 적용된 CSS3스타일 속성

p.id
p.innerHTML 
p.tagName
p.style.color
p.style.backgroundColor
p.style.font-size

#DOM객체들 사이의 관계
부모 , 자식, 형제의 관계로 연결되있다.



parentElement 프로퍼티 - 부모객체 
children 프로퍼티 - 직계 자식 객체들의 컬렉션.
firstElementChild 프로퍼티 - 첫번째 자식
lastElementChild 프로퍼티 - 마지막 자식
previousElementSibling 프로퍼티
nextElementSibling 프로퍼티

sibling은 DOM트리에서 동일한 Level에 있는
모든 사촌 및 형제 DOM객체를 지칭한다.
